3. Estándares de Codificación - JavaScript

3.1 Nomenclatura para Variables y Constantes

Los estándares de nomenclatura son esenciales para mantener un código legible y mantenible. A continuación, se detallan las convenciones para variables y constantes:

Variables

  • Utilizar camelCase para nombres de variables
  • Los nombres deben ser descriptivos y en inglés
  • Evitar nombres de una sola letra excepto en bucles o contextos específicos
// Correcto
let userName = 'Franco';
let totalAmount = 150.75;

// Incorrecto
let u = 'Franco';
let tamount = 150.75;

Constantes

  • Utilizar UPPERCASE con guiones bajos para constantes globales
  • Utilizar camelCase para constantes locales
// Constantes globales
const MAX_USERS = 100;
const API_BASE_URL = 'https://api.example.com';

// Constantes locales
function calculateArea() {
  const circleRadius = 5;
  return Math.PI * circleRadius * circleRadius;
}

3.2 Nomenclatura para Clases, Métodos y Comentarios

Clases

  • Utilizar PascalCase para nombres de clases
  • Los nombres deben ser sustantivos que describan la entidad
// Correcto
class UserProfile {
  // Propiedades y métodos
}

class DatabaseConnection {
  // Propiedades y métodos
}

Métodos y Funciones

  • Utilizar camelCase para nombres de métodos y funciones
  • Los nombres deben comenzar con un verbo que describa la acción
  • Mantener funciones cortas y enfocadas en una sola responsabilidad
// Correcto
function calculateTotal(items) {
  // Código
}

class ShoppingCart {
  addItem(item) {
    // Código
  }
  
  removeItem(itemId) {
    // Código
  }
}

Comentarios

  • Usar comentarios para explicar el "por qué", no el "qué"
  • Documentar todas las funciones con JSDoc
  • Mantener comentarios actualizados cuando el código cambie
/**
* Calcula el precio total incluyendo impuestos
* @param {number} basePrice - El precio base sin impuestos
* @param {number} taxRate - La tasa de impuesto (por ejemplo, 0.19 para 19%)
* @returns {number} El precio total con impuestos
*/
function calculateTotalPrice(basePrice, taxRate) {
  // Validamos que los parámetros sean números positivos
  if (basePrice < 0 || taxRate < 0) {
    throw new Error('Los valores deben ser positivos');
  }
  
  return basePrice * (1 + taxRate);
}

3.3 Estructura y Formato del Código

Indentación y Espaciado

  • Utilizar 2 espacios para la indentación
  • Añadir espacio después de palabras clave y antes de llaves
  • Añadir espacio alrededor de operadores
// Correcto
if (condition) {
  doSomething();
} else {
  doSomethingElse();
}

const sum = a + b;

Organización de archivos

  • Un componente o clase por archivo
  • Importaciones agrupadas y ordenadas: primero librerías externas, luego componentes internos
  • Exportar al final del archivo
// Importaciones externas
import React from 'react';
import { connect } from 'react-redux';

// Importaciones internas
import { Button } from '../components/Button';
import { formatDate } from '../utils/dateHelpers';

// Definición de componente/clase
class UserDashboard extends React.Component {
  // código del componente
}

// Exportación
export default connect(mapStateToProps)(UserDashboard);
