Nomenclatura para Variables y Constantes

Una guía completa sobre cómo nombrar correctamente variables y constantes en programación

¿Por qué es importante la nomenclatura?

Un buen código no solo debe funcionar correctamente sino que también debe ser legible y mantenible. La forma en que nombramos nuestras variables y constantes afecta directamente la legibilidad del código y la facilidad con la que otros desarrolladores (¡o tú mismo en el futuro!) pueden entenderlo.

Camel Case

La primera palabra comienza con minúscula y cada palabra siguiente comienza con mayúscula. No hay espacios ni guiones.

// Bien
let nombreUsuario = "Juan";
let edadUsuario = 25;
let contadorClics = 0;

Usado comúnmente en JavaScript, Java, TypeScript y muchos otros lenguajes para variables y funciones.

Pascal Case

Cada palabra comienza con mayúscula, incluyendo la primera. No hay espacios ni guiones.

// Bien
class UsuarioAdmin { }
interface ProductoDetalle { }
function CalcularTotal() { }

Usado generalmente para clases, interfaces y a veces para funciones en C#, TypeScript y otros lenguajes.

Snake Case

Todas las palabras en minúsculas, separadas por guiones bajos.

// Bien
let nombre_usuario = "María";
const max_intentos = 3;
function calcular_total() { }

Popular en Python, Ruby, y archivos de configuración. También se usa para constantes en algunos lenguajes.

Kebab Case

Todas las palabras en minúsculas, separadas por guiones.

// Bien
// En CSS
.user-profile { }
.nav-item { }
// En archivos: user-settings.js

Común en URLs, nombres de archivos, CSS y HTML.

Consejos para nombrar Variables y Constantes

1 Sé Descriptivo

Los nombres deben indicar claramente qué contiene la variable.

// Bien
let precioTotal = 299.99;
// Mal
let p = 299.99;

2 Constantes en Mayúsculas

Las constantes suelen nombrarse en mayúsculas con guiones bajos.

// Bien
const MAX_USUARIOS = 100;
const API_KEY = "abc123";

3 Evita Abreviaturas Ambiguas

Las abreviaturas pueden causar confusión. Sé explícito siempre que puedas.

// Bien
let cantidadDeElementos = 5;
// Mal
let cantElem = 5;

4 Consistencia

Mantén un estilo de nomenclatura consistente en todo tu código.

// Inconsistente (¡evitar!)
let nombreUsuario = "Ana";
let edad_usuario = 28;
let UserEmail = "ana@ejemplo.com";

5 Prefijos para Tipos

Algunos desarrolladores usan prefijos para indicar el tipo de variable.

// Ejemplo en algunos estilos
let strNombre = "Pedro"; // string
let intEdad = 30; // integer
let arrColores = ["rojo", "azul"]; // array

6 Booleanos con Prefijo

Para variables booleanas, usa prefijos como "is", "has", "can".

// Bien
let isActivo = true;
let hasPermiso = false;
let canEditar = true;

Errores Comunes a Evitar

⚠️

Nombres Demasiado Genéricos

Evita nombres como "data", "temp", "value" sin contexto adicional.

⚠️

Nombres Demasiado Largos

Aunque descriptivos, nombres excesivamente largos dificultan la lectura.

⚠️

Usar Números en Nombres

Evita nombres como "usuario1", "usuario2". Usa arrays o estructuras apropiadas.

⚠️

Nombres que Causan Confusión

Evita nombres como "l" (letra L) o "O" (letra O) que pueden confundirse con "1" y "0".

Ejemplo Completo

A continuación se muestra un ejemplo de código con buenas prácticas de nomenclatura:

// Constantes
const MAX_INTENTOS_LOGIN = 3;
const API_URL = "https://api.ejemplo.com";

// Clases
class UsuarioAdmin {
  constructor(nombre, email) {
    // Variables de instancia
    this.nombreCompleto = nombre;
    this.emailUsuario = email;
    this.isActivo = true;
  }
  
  // Métodos
  obtenerDatosUsuario() {
    return {
      nombre: this.nombreCompleto,
      email: this.emailUsuario
    };
  }
}

// Función
function calcularPrecioTotal(precioBase, impuestosPorcentaje) {
  const factorImpuesto = 1 + (impuestosPorcentaje / 100);
  return precioBase * factorImpuesto;
}
